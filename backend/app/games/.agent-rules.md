# CUR8 Games — Agent Rules

**Scope:** every game under `backend/app/games/<game_id>/`.
**Audience:** automated agents. Read top-to-bottom, then execute the checklist.

---

## 0. MUST / NEVER (read first)

**MUST**
- Use `snake_case` for `<game_id>` (folder + DB id). Allowed regex: `^[a-z0-9_]+$`.
- Ship ES6 modules (`type="module"`), one class per file.
- Integrate the Platform SDK via the documented API only (see §3).
- Read all numeric values from a single `js/config/GameConfig.js`.
- Target 60 FPS on mid-range mobile; use `deltaTime` (seconds) in update loops.
- Provide `thumbnail.png` (there is a script `backend/scripts/generate_thumbnail_previews.py`) (first a placeholder `.jpg`) at the game root. The thumbnail script generates the `.webp` preview.
- Register the game with a script under `backend/scripts/register_<game_id>.py` following the Blackjack template (see §4).
- If the game has server-side logic, place it in `backend/app/games/<game_id>_be/` (mirror naming).

**NEVER**
- Use globals, `var`, synchronous XHR, or `eval`.
- Touch reward / XP / quest / social logic from inside the game JS — emit SDK events only.
- Hardcode magic numbers outside `GameConfig`.
- Write to `localStorage` for gameplay state that should be server-authoritative (scores, coins, currency).
- Allocate inside the hot loop (no `new`, no array literals per frame). Use object pools.
- Block the game loop with `await` on platform calls — fire-and-forget or `await` only at boundaries (start/end).
- Manipulate the DOM inside `update()` / `render()`; update HUD on state-change only.
- Use `..` / `/` / `\` in any path passed to backend (game_id is path-validated).

## 3. Platform SDK — REAL API (only these methods exist)

Source of truth: [backend/sdk/platformsdk.d.ts](backend/sdk/platformsdk.d.ts). Do not invent methods.

| Method | When to call |
|---|---|
| `init(opts?)` | Once, before anything else. Await it. |
| `sendScore(score, meta?)` | On every score change (cheap, fire-and-forget). |
| `gameOver(finalScore, meta?)` | Once, on game over. `meta` may include `timePlayed`, `achievements[]`. |
| `levelCompleted(level, meta?)` | On level clear. `meta`: `timeToComplete`, `stars`, `perfectClear`. |
| `requestFullScreen()` / `toggleFullscreen()` / `isFullscreen()` | On user gesture only. Use `toggleFullscreen` for iOS. |
| `on(evt, cb)` / `off(evt, cb)` | Events: `start`, `pause`, `resume`, `exit`, `config`. |
| `getState()` | Read-only `{ score, level, isPaused }`. |

There is **no** `startSession`, `endSession`, or `trackEvent`. If you find references to those, they are wrong.

Achievements bonus XP (e.g. `natural_blackjack`) is delivered by passing the key in `gameOver({ achievements: ['natural_blackjack'] })`. The backend matches it against `XPRule` rows of type `achievement`.

## 4. Game Registration Script — required

Every new game ships with `backend/scripts/register_<game_id>.py`. Use [register_blackjack.py](backend/scripts/register_blackjack.py) as the template. Constants to define at top of file:

- `GAME_ID` — must equal the folder name.
- `GAME_DATA` — dict matching the `Game` model. Required keys: `game_id`, `title`, `description`, `author`, `version`, `thumbnail`, `entry_point` (`"index.html"`), `category`, `tags` (JSON string), `status_id`, `steem_rewards_enabled`, `extra_data` (JSON string).
- `XP_RULES` — list of `XPRule` dicts. Supported `rule_type`: `flat`, `threshold`, `score_multiplier`, `achievement`, `high_score_bonus`. `parameters` is a JSON string.
- `QUESTS` — list of `Quest` dicts. `config` is a JSON string and must include `game_id`, `type`, `category`, `reset_period`, `reset_on_complete`.

The script MUST:
- Accept `--force` to overwrite.
- Use `get_db_session()` context manager.
- On `--force`: delete existing `Game` + its `XPRule` rows before re-insert.
- Commit once at the end; rollback on exception; print a final summary.

Do not duplicate the helper logic — copy the Blackjack script structure verbatim and only change the data blocks.

---

## 5. Performance Budget

- Frame budget: **≤ 16 ms** (60 FPS). Profile with DevTools Performance, not `console.time` in the loop.
- Per-frame allocations: **0** in steady state. Pool bullets, particles, enemies.
- Asset weight per game: **≤ 2 MB** total (sprites + audio). Use WebP for static images, sprite sheets for animations, OGG/MP3 ≤ 500 KB per SFX.
- DOM writes: only on state change, never every frame.
- `requestAnimationFrame` only — no `setInterval` for the loop.

---

## 6. Code Quality (SonarQube-friendly)

- One class per file, file name = class name.
- Cyclomatic complexity per function: **≤ 10**.
- Function length: **≤ 40 lines**. Split otherwise.
- No duplicated blocks ≥ 6 lines — extract.
- Always handle Promise rejection (`try/catch` on `await`, `.catch()` on fire-and-forget).
- Public methods: JSDoc with `@param` / `@returns` types.
- Use `const` by default, `let` only when reassigned, never `var`.

## PATTERN OBBLIGATORI

### 1. Game Class Pattern
```javascript
// js/core/Game.js
export class Game {
    constructor(canvas) {
        this.canvas = canvas;
        this.ctx = canvas.getContext('2d');
        this.running = false;
        this.lastTime = 0;
    }

    start() {
        this.running = true;
        requestAnimationFrame(this.gameLoop.bind(this));
    }

    gameLoop(timestamp) {
        if (!this.running) return;
        
        const deltaTime = (timestamp - this.lastTime) / 1000;
        this.lastTime = timestamp;
        
        this.update(deltaTime);
        this.render();
        
        requestAnimationFrame(this.gameLoop.bind(this));
    }

    update(dt) {
        // Update logic
    }

    render() {
        // Render logic
    }
}
```

### 2. Entity Pattern
```javascript
// js/entities/Entity.js
export class Entity {
    constructor(x, y, width, height) {
        this.x = x;
        this.y = y;
        this.width = width;
        this.height = height;
        this.velocityX = 0;
        this.velocityY = 0;
    }

    update(dt) {
        this.x += this.velocityX * dt;
        this.y += this.velocityY * dt;
    }

    render(ctx) {
        ctx.fillRect(this.x, this.y, this.width, this.height);
    }

    getBounds() {
        return {
            left: this.x,
            right: this.x + this.width,
            top: this.y,
            bottom: this.y + this.height
        };
    }
}
```

## INTEGRAZIONE PLATFORM SDK

### Setup SDK
```javascript
// js/platform/PlatformIntegration.js
import PlatformSDK from '/sdk/platformsdk.min.js';

export class PlatformIntegration {
    constructor() {
        this.sdk = new PlatformSDK();
        this.sessionActive = false;
    }

    async startSession() {
        if (this.sessionActive) return;
        
        try {
            await this.sdk.startSession();
            this.sessionActive = true;
        } catch (error) {
            console.error('Failed to start session:', error);
        }
    }

    async endSession(score, data = {}) {
        if (!this.sessionActive) return;
        
        try {
            await this.sdk.endSession(score, data);
            this.sessionActive = false;
        } catch (error) {
            console.error('Failed to end session:', error);
        }
    }

    async trackEvent(eventType, data = {}) {
        try {
            await this.sdk.trackEvent(eventType, data);
        } catch (error) {
            console.error('Failed to track event:', error);
        }
    }
}
```

### Uso in Game
```javascript
import { PlatformIntegration } from './platform/PlatformIntegration.js';

export class Game {
    constructor(canvas) {
        this.canvas = canvas;
        this.platform = new PlatformIntegration();
        this.score = 0;
    }

    async start() {
        await this.platform.startSession();
        this.running = true;
        requestAnimationFrame(this.gameLoop.bind(this));
    }

    async gameOver() {
        this.running = false;
        await this.platform.endSession(this.score, {
            level: this.currentLevel,
            time: this.elapsedTime
        });
    }
}
```

## PERFORMANCE RULES

### 1. Canvas Rendering
```javascript
// ✅ GOOD - Clear and render efficiently
render() {
    this.ctx.clearRect(0, 0, this.canvas.width, this.canvas.height);
    
    // Render only visible entities
    this.entities
        .filter(e => this.isVisible(e))
        .forEach(e => e.render(this.ctx));
}

// ❌ BAD - Multiple clears, inefficient
render() {
    this.ctx.fillStyle = 'black';
    this.ctx.fillRect(0, 0, this.canvas.width, this.canvas.height);
    this.entities.forEach(e => {
        this.ctx.clearRect(e.x, e.y, e.width, e.height);
        e.render(this.ctx);
    });
}
```

### 2. Object Pooling
```javascript
// ✅ GOOD - Reuse objects
class BulletPool {
    constructor(size) {
        this.pool = [];
        this.active = [];
        
        for (let i = 0; i < size; i++) {
            this.pool.push(new Bullet());
        }
    }

    spawn(x, y) {
        const bullet = this.pool.pop();
        if (bullet) {
            bullet.reset(x, y);
            this.active.push(bullet);
        }
    }

    recycle(bullet) {
        const index = this.active.indexOf(bullet);
        if (index > -1) {
            this.active.splice(index, 1);
            this.pool.push(bullet);
        }
    }
}

// ❌ BAD - Create new objects every frame
update() {
    if (this.shootCooldown <= 0) {
        this.bullets.push(new Bullet(this.x, this.y)); // Memory leak!
    }
}
```

### 3. Avoid Memory Leaks
```javascript
// ✅ GOOD - Clean up
destroy() {
    this.entities.forEach(e => e.destroy());
    this.entities = [];
    
    window.removeEventListener('keydown', this.keyHandler);
    window.removeEventListener('keyup', this.keyHandler);
    
    this.canvas = null;
    this.ctx = null;
}

// ❌ BAD - No cleanup
destroy() {
    // Listeners still active!
    // References still held!
}
```

## COMMON PATTERNS

### State Machine
```javascript
export class GameStateMachine {
    constructor() {
        this.states = {
            'menu': new MenuState(),
            'playing': new PlayingState(),
            'paused': new PausedState(),
            'gameover': new GameOverState()
        };
        this.currentState = 'menu';
    }

    setState(stateName) {
        this.states[this.currentState].exit();
        this.currentState = stateName;
        this.states[this.currentState].enter();
    }

    update(dt) {
        this.states[this.currentState].update(dt);
    }

    render(ctx) {
        this.states[this.currentState].render(ctx);
    }
}
```

### Collision Detection
```javascript
export function checkCollision(a, b) {
    return a.x < b.x + b.width &&
           a.x + a.width > b.x &&
           a.y < b.y + b.height &&
           a.y + a.height > b.y;
}

export function checkCircleCollision(a, b) {
    const dx = a.x - b.x;
    const dy = a.y - b.y;
    const distance = Math.sqrt(dx * dx + dy * dy);
    return distance < a.radius + b.radius;
}
```

## FORBIDDEN

### ❌ NO Global Variables
```javascript
// BAD
let globalScore = 0;
let globalPlayer = null;

// GOOD
```

### ❌ NO Blocking Operations
```javascript
// BAD
function loadImageSync(url) {
    const xhr = new XMLHttpRequest();
    xhr.open('GET', url, false); // Synchronous!
    xhr.send();
    return xhr.response;
}

// GOOD
async function loadImage(url) {
    return new Promise((resolve, reject) => {
        const img = new Image();
        img.onload = () => resolve(img);
        img.onerror = reject;
        img.src = url;
    });
}
```

### ❌ NO Direct DOM Manipulation in Game Loop
```javascript
// BAD
update() {
    document.getElementById('score').textContent = this.score;
}

// GOOD - Update UI outside game loop
updateUI() {
    document.getElementById('score').textContent = this.score;
}

gameLoop() {
    this.update(dt);
    this.render();
    
    if (this.scoreChanged) {
        this.updateUI();
        this.scoreChanged = false;
    }
}
```
